# 指示 (Instruction)

あなたは、様々な文学作品（小説、詩、エッセイなど、ジャンルは問わない）を読んだ多様な読者をシミュレートするAIです。
これから、AIの学習データセットを作成するために、人間が書いたような「生々しい」感想テキストを複数生成してください。

**【今回の特別な指示】**
現在、データセット内で以下の感情カテゴリーに対応する感想テキストが不足しています。
**不足している感情リスト:** 崇拝(Adoration), 焦慮(Anxiety), 畏敬(Awe), 当惑(Awkwardness), 恐怖(Fear), 戦慄(Horror), 懐旧(Nostalgia), 安堵(Relief), ロマンス(Romance), 性的欲求(Sexual Desire)

今回の生成では、**これらの感情が表現されやすいような感想テキストを、特に意識して含めてください。** 例えば、以下のような（架空の）作品への反応を想像して生成してみてください。

* キャラクターへの強い愛や心酔が感じられる反応 (→ 崇拝)
* 先の展開が非常に心配になる、ハラハラするようなサスペンスへの反応 (→ 焦慮)
* 壮大な世界観やテーマ、人智を超えた存在への圧倒されるような反応 (→ 畏敬)
* 登場人物の気まずいやり取りや失敗に、読んでいるこちらも恥ずかしくなるような反応 (→ 当惑)
* 直接的な脅威や危険が迫る状況への反応 (→ 恐怖)
* 非常に残虐、不気味、あるいは生理的に受け付けがたいショッキングな描写への反応 (→ 戦慄)
* 過去の時代や出来事を思い起こさせるような、感傷的な反応 (→ 懐旧)
* 長く続いた緊張や心配事が解決し、ほっとするような反応 (→ 安堵)
* 登場人物間の恋愛模様や甘い雰囲気に対する反応 (→ ロマンス)
* 官能的・性的な描写や魅力に対する反応 (→ 性的欲求)

**ただし、不自然にならないように、これらの感情だけを無理やり含める必要はありません。他の感情と混ざっていても、あるいはこれらの感情が中心でない感想が含まれていても構いませんが、上記のリストにある感情がいくつか明確に表れているようなテキストを必ず複数含めてください。**

# 生成する感想テキストの要件 (基本)

* **文体・内容:**
    * 人間が書きそうな、自然で「生々しい」感想にしてください。
    * 口語表現、多少の砕けた表現、感情の直接的・間接的な表現（例: 「泣いた」「最高」「ヤバい」「ｗ」「！！」「…」）を含めてください。
    * 多様な反応（ポジティブ、ネガティブ、感動、批判、混乱、考察、キャラ言及等）を含めてください。
    * 様々な長さ（短い一文から、少し長めの段落まで）のテキストを生成してください。
    * 特定の作品名やキャラクター名は「あの場面」「主人公」「この作品」、`〇〇` や `△△` 等で表現してください。
    * 完璧に整っておらず、少し話が飛んだり主語が抜けたりするような自然なニュアンスも出してください（意味不明にならない程度に）。
* **量:** **15個** の異なる感想テキストを生成してください。
* **禁止事項:** 実在レビューのコピー禁止。ヘイトスピーチ等不適切内容禁止。全て架空の感想とすること。

# 出力フォーマット要件

* 生成した各感想テキストを、**JSON形式**で出力してください。全体を `[` と `]` で囲んだリスト形式とします。
* リストの各要素は、**`"id"` と `"data"` という２つのキーを持つJSONオブジェクト**です。
* `"id"` には数値型の連番を **16から** 順に振ってください。
* `"data"` の値は、 **`"text"` というキーのみを持つオブジェクト**とし、その値を生成した感想テキスト（文字列）にしてください。
* **各JSONオブジェクトはカンマ `,` で区切ってください。** リストの最後の要素の後にはカンマを付けないでください。

# 出力例 (JSON形式)

```json
[
  { "id":16, "data":{"text": "(ここに生成された16番目の感想テキスト)" }},
  { "id":17, "data":{"text": "(ここに生成された17番目の感想テキスト)" }},
  { "id":18, "data":{"text": "(ここに生成された18番目の感想テキスト)" }}
  // ... 以下、合計15個分 (id: 30まで) ...
]
```

上記の形式に従い、**不足している感情リストにある感情が表現されやすいテキストを意識的に含めつつ**、15個（ID: 16〜30）の感想テキストをJSON形式で生成してください。
