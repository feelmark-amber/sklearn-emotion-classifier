# 指示 (Instruction)

あなたは、様々な文学作品（小説、詩、エッセイなど、ジャンルは問わない）を読んだ多様な読者をシミュレートするAIです。
これから、AIの学習データセットを作成するために、人間が書いたような「生々しい」感想テキストを複数生成してください。**前回までの生成で、同じペルソナから似たような文章が出力される傾向がありました。今回は特に、各感想の独自性と表現の多様性を最大限に高めるようにしてください。**

**【重要：毎回異なる多様な出力を得るための指示】**

**ステップ1: 詳細な反応シナリオの考案**
まず、以下の読者ペルソナのリストから **少なくとも5種類以上**、そして **合計15人分** の読者像（重複可、組み合わせも可）を内部的に想定してください。
* **読者ペルソナリスト:** [熱心な考察好き、辛口批評家、キャラクターに感情移入しやすいタイプ、純粋に楽しむエンタメ読者、斜に構えた読者、昔を懐かしむ年配者、流行に敏感な若者、情景描写を重視する読者、哲学的な思索を好むタイプ]
次に、それぞれの読者が **異なるタイプの作品**（例: SF、恋愛、歴史、ミステリ、純文学、ホラー、エッセイなど）を読み、**具体的な（架空の）作品のディテール**（例：予想外の結末、読者の期待を裏切るキャラ行動、美しいが難解な比喩、特定の社会問題への言及、強烈な印象を残す脇役など）に触れたと仮定します。その上で、**前回までとは異なる、読者の個人的な感情や思考が揺れ動くような、ユニークな反応シナリオ**を15個、より詳細に考えてください。

**ステップ2: 感想テキストの生成**
上記ステップ1で考案した15の「詳細な反応シナリオ」に基づき、感想テキストを生成します。
* **ペルソナへの固執回避:** 想定したペルソナはあくまで**出発点**とし、**それに固執しすぎず**、考案したシナリオと感情のニュアンスを**最も自然で、かつ独自性のある言葉**で表現することに重点を置いてください。同じペルソナ設定でも、シナリオや感情の機微によって全く異なる口調・意見・表現になるはずです。
* **表現スタイルの多様化:** 生成する15個の中に、**[1]感情的な言葉を多用する感想、[2]冷静に分析・評価する感想、[3]比喩や情景描写で感情を示唆する感想、[4]他の作品や自身の体験と比較する感想、[5]自問自答したり、読者に問いかけたりするような感想**、といった**異なる表現スタイル**のものを意識的に含めてください。
* **独自性の徹底:** 生成する15個の感想は、互いに内容、視点、文体、感情のニュアンスが**明確に異なる**ようにしてください。**過去の生成物や、ありきたりな定型表現（例：「鳥肌が立った」「一気読みした」「考えさせられた」「伏線回収が見事」「世界観がすごい」「神作品」「推せる」など）は、徹底的に避けてください。** 新しい言葉や言い回しを試してください。
* **多様な感情表現:** 感情カテゴリー名（例：「崇拝」「焦慮」など）を直接使わず、より具体的で多様な言い回し（例：「もはや信仰に近い」「心臓掴まれてる感じ」「魂が震えた」「いたたまれない」「足がすくむ」「吐き気を催す」「胸が締め付けられる」「肩の荷が下りた」「頬が緩む」「妙に生々しい」など）を使ってください。

# 生成する感想テキストの要件 (基本)

* **文体・内容:**
    * 人間が書きそうな、自然で「生々しい」感想であること。
    * 口語表現、多少の砕けた表現、感情の直接的・間接的な表現（例: 「泣いた」「最高」「ヤバい」「ｗ」「！！」「…」）を含むこと。
    * ポジティブ、ネガティブ、両方の混在、感動、批判、混乱、考察、キャラクターへの言及など、**幅広い**反応を含むこと。
    * 様々な長さ（短い一文から、少し長めの段落まで）であること。
    * 特定の作品名やキャラクター名は「あの場面」「主人公」「この作品」、`〇〇` や `△△` 等で表現すること。
    * 完璧に整っておらず、少し話が飛んだり主語が抜けたりするような自然なニュアンスも出すこと（意味不明にならない程度に）。
* **量:** **15個** の異なる感想テキストを生成すること。
* **禁止事項:** 実在レビューのコピー禁止。ヘイトスピーチ等不適切内容禁止。全て架空の感想とすること。**過去の生成物と酷似しないこと。**

# 出力フォーマット要件

* 生成した各感想テキストを、**JSON形式**で出力してください。全体を `[` と `]` で囲んだリスト形式とします。
* リストの各要素は、**`"id"` と `"data"` という２つのキーを持つJSONオブジェクト**です。
* `"id"` には数値型の連番を **（【★ここに前回の最後のIDの次の番号を指定してください★】例: 121）** から順に振ってください。
* `"data"` の値は、 **`"text"` というキーのみを持つオブジェクト**とし、その値を生成した感想テキスト（文字列）にしてください。
* **各JSONオブジェクトはカンマ `,` で区切ってください。** リストの最後の要素の後にはカンマを付けないでください。

# 出力例 (JSON形式) - 形式参考用

```json
[
  { "id": (開始ID), "data":{"text": "(ここに生成された感想テキスト)" }},
  { "id": (開始ID+1), "data":{"text": "(ここに生成された感想テキスト)" }}
  // ... 以下、合計15個分 ...
]
```

上記の形式に従い、**毎回異なる多様な読者像と詳細な反応シナリオを内部で考案し、それに基づいて、ペルソナに固執せず、異なる表現スタイルを盛り込み、過去の生成物やありきたりな表現を避け、独自の言い回しで書かれた生々しい感想テキスト**を15個生成してください。**(※IDはこちらで指定しますので、【★開始ID★】から始めてください)**
