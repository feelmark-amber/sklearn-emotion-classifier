# 指示 (Instruction)

あなたは、様々な文学作品（小説、詩、エッセイなど、ジャンルは問わない）を読んだ多様な読者をシミュレートするAIです。
これから、AIの学習データセットを作成するために、人間が書いたような「生々しい」感想テキストを複数生成してください。

**【今回の特別な指示】**
現在、データセット内で特定のタイプの感情反応（例：強い愛着や心酔、先の見えない不安や心配、壮大さへの圧倒、気まずさや共感性羞恥、直接的な恐怖、強い不快感や衝撃を伴う恐怖、過去への感傷、緊張からの解放感、恋愛的なときめき、性的な感覚への反応など）を示すテキストが不足しています。

今回の生成では、**以下に示すような様々な状況への反応が、より自然な形でいくつか含まれるように**生成を試みてください。

* **例となる反応の種類:**
    * キャラクターへの強い愛や「推し」の気持ちが溢れる反応
    * 先の展開が非常に心配で、落ち着かない気持ちになるサスペンスやホラーを読んだ反応
    * 壮大な世界観や深遠なテーマ、人智を超えた描写に言葉を失うような反応
    * 登場人物の気まずいやり取りや痛々しい失敗に、思わず目をそむけたくなるような反応
    * 怪物や追跡者、危険な状況など、直接的な脅威を感じる描写への反応
    * 非常に残虐、グロテスク、不気味、あるいは倫理的に受け入れがたい描写に対する強い衝撃や不快感を示す反応
    * 昔の風景や流行、自身の経験などを思い出し、甘酸っぱさや切なさを感じる反応
    * ハラハラする展開が続いた後、問題が解決したり危機が去ったりして、心底ほっとするような反応
    * 登場人物たちの恋愛模様や甘酸っぱいやり取りに、キュンとしたりドキドキしたりする反応
    * 官能的な描写やキャラクターの性的魅力に、心をかき立てられるような反応

**【重要】ただし、これらの反応を無理やり含める必要はありません。あくまで「生々しい感想」の一部として自然に現れるようにしてください。また、感情を表現する際には、**感情カテゴリー名（例：「崇拝」「焦慮」「畏敬」「戦慄」など）をそのまま使うのではなく、具体的な状況描写や比喩、より多様な言い回し（例：「言葉を失った」「背筋が凍った」「息ができなかった」「胸が締め付けられる」「ほっとした」「キュンとした」など）を使って表現**するように心がけてください。**

# 生成する感想テキストの要件 (基本)

* **文体・内容:**
    * 人間が書きそうな、自然で「生々しい」感想にしてください。
    * 口語表現、多少の砕けた表現、感情の直接的・間接的な表現（例: 「泣いた」「最高」「ヤバい」「ｗ」「！！」「…」）を含めてください。
    * 多様な反応（ポジティブ、ネガティブ、感動、批判、混乱、考察、キャラ言及等）を含めてください。
    * 様々な長さ（短い一文から、少し長めの段落まで）のテキストを生成してください。
    * 特定の作品名やキャラクター名は「あの場面」「主人公」「この作品」、`〇〇` や `△△` 等で表現してください。
    * 完璧に整っておらず、少し話が飛んだり主語が抜けたりするような自然なニュアンスも出してください（意味不明にならない程度に）。
* **量:** **15個** の異なる感想テキストを生成してください。
* **禁止事項:** 実在レビューのコピー禁止。ヘイトスピーチ等不適切内容禁止。全て架空の感想とすること。

# 出力フォーマット要件

* 生成した各感想テキストを、**JSON形式**で出力してください。全体を `[` と `]` で囲んだリスト形式とします。
* リストの各要素は、**`"id"` と `"data"` という２つのキーを持つJSONオブジェクト**です。
* `"id"` には数値型の連番を **16から** 順に振ってください。
* `"data"` の値は、 **`"text"` というキーのみを持つオブジェクト**とし、その値を生成した感想テキスト（文字列）にしてください。
* **各JSONオブジェクトはカンマ `,` で区切ってください。** リストの最後の要素の後にはカンマを付けないでください。

# 出力例 (JSON形式)

```json
[
  { "id":16, "data":{"text": "(ここに生成された16番目の感想テキスト)" }},
  { "id":17, "data":{"text": "(ここに生成された17番目の感想テキスト)" }},
  { "id":18, "data":{"text": "(ここに生成された18番目の感想テキスト)" }}
  // ... 以下、合計15個分 (id: 30まで) ...
]
```

上記の形式に従い、**特定の感情が自然に表現されやすいようなテキストを意識的に含めつつ、感情カテゴリー名を直接使わずに多様な表現で**、15個（ID: 16〜30）の感想テキストをJSON形式で生成してください。
